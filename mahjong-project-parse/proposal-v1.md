# 方案 V1

## 方案定位

以 `code-back/mahjong` 为原始后端项目，先建立一套可学习、可协作、可继续开发、可验收的项目知识包。第一版不修改业务代码，而是通过文档、索引、迁移计划和验收契约降低后续协作成本。

## MVP 划分（已与用户确认）

第一阶段 MVP 包含：

- F1 项目画像与目录结构解析。
- F2 Go 包、入口、模块职责和依赖关系解析。
- F3 麻将核心流程解析，覆盖房间、用户、牌局、匹配、RPC、异常/恢复。
- F4 新需求影响范围表。
- F5 文档同步策略，将链路产物沉淀到 `document/`。
- F6 轻量扫描/索引脚本。
- F7 验收契约与检查清单。

后置范围：

- 业务代码迁移。
- 麻将核心逻辑重构。
- 前端业务功能开发。
- 复杂自动化文档生成。

MVP 完成定义：完成 F1、F2、F3、F4、F5、F6、F7 即第一版完成。

## 目录策略

- 链路标准产物放在 `mahjong-project-parse/output/`。
- 面向项目使用的最终文档同步到 `document/`。
- 对话和关键决策记录放在 `command/`。
- 轻量脚本放在 `script/`。
- `code-back/mahjong` 保持原始状态，不在第一版中修改。

## 验收策略

第一版验收采用四件套：

- 文档完整性。
- 源码引用。
- 影响范围表。
- 验收契约。
