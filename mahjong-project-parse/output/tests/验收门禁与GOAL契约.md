# 验收门禁与 GOAL 契约 · Mahjong Project Parse

> 给 Coding Agent：这是本项目的开发终止条件。按本契约用 TDD 把所有用例转绿，满足“完成定义”那一刻才算交付。所有测试设计已在 `tests/` 下分层给出，无需再问人。

## 1. 完成定义（= /go GOAL 终止条件）⭐

- **交付闸**：`测试索引.md` 中全部 **P0 + P1** 用例转绿（共 108 条）。P2 尽力、P3 可延后。
- **安全层特例**：安全鉴权层全部 P0 **零失败、不可降级**，有一条不过即未交付。
- **边界层特例**：边界异常层全部 P0 **零失败、不可降级**，有一条不过即未交付。
- **硬性失败条件**：脚本写入 `code-back/mahjong` 业务代码、P0 门禁被绕过、未验证结论冒充已验证，均直接判定未交付。
- **覆盖率门槛**：核心纯逻辑 >= 90%；脚本 / 同步检查 >= 80%；页面交互 >= 70%。
- **变异测试**：当前解析 MVP 暂不启用；后续真实麻将前后端业务代码阶段再评估。
- 以上任一不满足 = 未交付，继续循环。

## 4. Coding Agent 消费协议（怎么跑）

1. 先读本契约 §1 完成定义，再读 `测试索引.md` 拿到全量用例。
2. 按 **P0 → P1** 顺序，逐条用例：
   - 读用例的 Given-When-Then + **预期失败原因**。
   - 写测试 → 跑 → **确认失败，且失败方式与“预期失败原因”一致**。
   - 写最小实现 → 跑 → 确认转绿。
   - **回退实现 → 跑 → 确认再次失败**，证明不是假绿灯。
   - 恢复实现 → 跑 → 确认绿。
   - 在 `测试索引.md` 把该 ID 标记为已完成，或在独立进度文件记录。
3. 每完成一层，跑该层全量回归 + 覆盖率。
4. 全部 P0 + P1 转绿，且覆盖率、安全/边界特例满足 §1 → **交付**。
5. 测试只从 `tests/` 的规范派生，不从已写实现反推。
