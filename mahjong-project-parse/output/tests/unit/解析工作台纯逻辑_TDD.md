## 2. 测试策略

| 优先级 | 测试维度 | 依据来源 | 理由 |
| :--- | :--- | :--- | :--- |
| P0 | 文档索引、源码引用、AC 状态、脚本越界 | PRD F5-F7 / 蓝图 L1 | 直接决定能否交付 |
| P1 | 完成度、风险等级、未知状态降级 | 页面文档状态与边界 | 决定页面展示可信度 |
| P2 | 统计展示和排序稳定性 | 设计蓝图 §5 | 改善阅读体验 |

### 2.2 不测什么

- 不测 `code-back/mahjong` 的房间、用户、牌局、匹配、RPC 等 Go 业务逻辑。
- 不测 HTTP API、数据库或登录态。

## 3. 测试用例清单

| ID | 用例名称 | 优先级 | Given | When | Then | 预期失败原因 | 来源 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| U-WB-01 | 解析合法文档索引 | P0 | 存在包含文档、页面、状态的 JSON 索引 | 调用索引解析逻辑 | 返回结构化条目，包含标题、路径、类型、状态 | 解析器未实现时返回空数组或字段丢失 | 测试决策蓝图 L1 |
| U-WB-02 | 缺少必填索引字段时报错 | P0 | 索引条目缺 `path` 或 `status` | 调用索引校验逻辑 | 返回字段缺失错误，不进入成功状态 | 校验缺失时错误索引被当作有效数据 | 技术方案 §5.1 |
| U-WB-03 | 计算 F1-F7 完成度 | P0 | 给定 F1-F7 的完成/待补充状态 | 调用完成度统计 | 输出总数、已完成数、百分比和阻塞项 | 统计未实现时完成度为 0 或错误百分比 | PRD §0.3 |

| U-WB-05 | 风险等级由缺口推导 | P1 | 存在 P0 缺源码引用、脚本失败、同步过期 | 调用风险等级计算 | 输出高风险，说明触发原因 | 风险计算未实现时高风险被降为低风险 | 页面文档：总览页 §7 |
| U-WB-06 | AC 状态按 P0/P1/P2 聚合 | P0 | AC 列表包含通过、待补充、阻塞 | 调用 AC 聚合逻辑 | 输出各优先级通过/未通过数量和总门禁状态 | 聚合未实现时 P0 阻塞不影响门禁 | 页面文档：验收契约页 §8 |
| U-WB-07 | 脚本越界写业务代码被拒 | P0 | 脚本声明写入 `code-back/mahjong` | 调用脚本安全边界判断 | 返回越界风险，门禁失败 | 判断未实现时越界脚本被标为可用 | PRD FR-6 / 蓝图 L5 |
| U-WB-08 | 未知状态降级为待确认 | P1 | 文档状态为未知值 `half-done` | 调用状态规范化 | 输出待确认，不冒充成功 | 规范化缺失时未知状态被吞掉或显示成功 | 蓝图 L6 |

## 4. Mock 策略

| 依赖 | Mock 方式 | 理由 | 关联用例 |
| :--- | :--- | :--- | :--- |
| 本地 JSON 索引 | 使用内存对象模拟合法/非法索引 | 单元层不读真实文件系统 | U-WB-01, U-WB-02 |
| 文档/脚本状态 | 使用最小结构化 fixture | 只验证纯逻辑 | U-WB-03 至 U-WB-08 |
