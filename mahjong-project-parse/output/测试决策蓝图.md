## 2 各层测试重点

> 以下内容将在阶段 I 后续轮次逐层确认。

### L1 单元

- 确认选择：1A，测解析工作台纯逻辑。
- 测试重点：
  - 文档索引解析：从 Markdown / JSON 索引读取文档条目、页面条目、状态条目。
  - 完成度统计：计算 F1-F7、页面文档、HTML 原型、设计产物、TDD 产物的完成状态。
  - 源码引用覆盖率：识别结论是否带 `code-back/mahjong` 路径引用，缺引用时标记未验证。
  - 风险等级计算：根据未验证结论、缺失文档、脚本失败、源码引用缺口生成高/中/低风险。
  - AC 状态计算：按 P0/P1/P2、通过/待补充/阻塞聚合验收门禁状态。
  - 脚本安全边界判断：识别脚本只读、只写索引/文档产物、越界写业务代码等状态。
- 明确不测：
  - 不测试 `code-back/mahjong` 的真实麻将业务领域逻辑。
  - 不测试房间、用户、牌局、匹配、RPC 的 Go 实现正确性。
  - 不测试数据库、后端 API 或登录态，因为解析工作台 MVP 不包含这些服务。
- 取舍理由：当前 TDD 契约服务于第一版解析链路，目标是保障文档、索引、页面和验收门禁可靠；真实麻将前后端本体将在后续产品链路中单独建立 TDD。

### L2 组件

- 确认选择：1A，测 6 个 MVP 页面组件状态与交互。
- 测试重点：
  - 总览页：状态指标、风险卡片、最近文档、下一步入口、索引缺失错误态。
  - 模块地图页：模块树、模块详情、文件族分组、源码引用、影响标签、搜索和筛选。
  - 核心流程页：流程选择器、流程图、涉及模块、验收点、异常路径、未验证状态。
  - 影响范围页：需求类型筛选、影响范围表、风险等级、推荐阅读、无匹配提示。
  - 脚本索引页：脚本卡片、输入输出、安全边界、运行结果、失败说明、复制脚本路径。
  - 验收契约页：完成定义、门禁状态、P0/P1/P2 AC、未验证结论、进入 TDD 入口。
- 每页通用覆盖：
  - 初始、加载、空状态、失败、异常、成功状态。
  - 搜索、筛选、复制路径、页面跳转、状态标签。
  - 移动端可读和长文本不挤压布局。
- 取舍理由：页面文档已经给出 6 个 MVP 页面及其状态、流转、验收标准；逐页测能保证页面验收与 PRD F1-F7 对齐。

### L3 集成契约

- 确认选择：1A，测文件与数据契约。
- 测试重点：
  - 页面清单契约：`页面清单.md` 中阶段一 6 个 MVP 页面必须都有对应 `pages/*.md` 和 `pages/*.html`。
  - 页面文档契约：每个页面文档必须包含元信息、页面概述、信息架构、交互流程、状态边界、页面流转、数据需求、验收标准。
  - HTML 原型契约：每个页面 HTML 必须能与页面文档一一对应，包含导航、主标题、核心区域和跨页链接。
  - 设计系统契约：页面 HTML 与 `tokens.css` / `DESIGN.md` 的关键 token、状态色、间距和组件命名保持一致。
  - 文档索引契约：Markdown / JSON 索引字段需要支撑页面搜索、筛选、状态标签、源码引用和验收状态。
  - 脚本输出契约：扫描/索引/同步检查脚本输出必须能被页面或验收文档引用，不写入 `code-back/mahjong` 业务代码。
  - 同步契约：`output/` 是链路产物源，`document/` 是最终阅读视图；同步检查需要识别缺失、过期和命名不一致。
- 明确不测：
  - 不测试 HTTP API、数据库 schema 或真实麻将前后端接口。
  - 不提前设计房间、匹配、牌局等真实游戏 API 契约。
- 取舍理由：解析工作台 MVP 的集成面是文件、文档、HTML、索引和脚本输出之间的契约；当前没有线上 API，文件级契约就是最真实的集成层。

### L4 E2E 业务流

- 确认选择：1A，测 4 条核心用户旅程。
- 测试重点：
  - lei 学习源码结构：总览页 → 模块地图页 → 核心流程页，验证能从项目状态进入模块结构，再进入核心流程理解。
  - he 判断需求影响：总览页 / 模块地图页 → 影响范围页 → 核心流程页，验证能按需求类型找到影响模块、流程、风险等级和推荐阅读。
  - 交付验收：总览页 → 验收契约页 → 定位未通过项，验证 P0/P1/P2 AC、未验证结论和门禁状态能支撑交付判断。
  - 脚本辅助：脚本索引页 → 复制命令/路径 → 验收契约页，验证脚本用途、输入输出、安全边界和验收检查能串起来。
- 每条旅程通用覆盖：
  - 从入口页进入目标页的导航链路。
  - 状态标签、源码引用、风险等级和验收项在跨页过程中保持一致。
  - 缺失数据或未验证结论不会被静默跳过。
- 明确不测：
  - 不测试真实麻将游戏流程，例如建房、匹配、出牌、结算。
  - 不测试真实用户登录、支付或在线对局。
- 取舍理由：解析工作台 MVP 的 E2E 价值是让 lei/he/Coding Agent 能沿文档链路完成理解、影响判断、脚本辅助和验收交接。

### L5 安全鉴权

- 确认选择：1A，测无登录场景的安全边界。
- 测试重点：
  - 无假登录/假权限：页面不展示登录态、管理员态、角色越权等不存在的能力。
  - 全员同内容：无登录前提下，所有页面内容一致，不因角色标签隐藏关键风险或门禁。
  - 未验证不可冒充已验证：缺源码引用、缺脚本输出、缺影响范围时必须显示未验证或阻塞。
  - 脚本不得写业务代码：脚本默认只读 `code-back/mahjong`，可写范围仅限索引、文档产物或日志。
  - 敏感信息保护：不展示真实用户隐私、不泄露环境变量、不复制大段源码。
  - P0 门禁不可绕过：验收契约中 P0 未通过时，不得显示已完成或可交付。
  - 断链可发现：页面链接、源码引用、文档引用失效时必须暴露，不静默吞掉。
- 明确不测：
  - 不测试玩家登录、房间权限、牌局越权、会话 token、RBAC。
  - 不测试真实麻将线上服务的鉴权体系。
- 取舍理由：解析工作台 MVP 不做账号体系，但仍必须守住只读源码、未验证显式标记、敏感数据不外泄、门禁不可绕过这些安全边界。

### L6 边界异常

- 确认选择：1A，测本地文档 / 索引 / 页面 / 脚本 / 同步的异常边界。
- 测试重点：
  - 索引缺失：本地 JSON 索引不存在、字段缺失、格式错误时，页面显示错误块并引导到脚本索引或验收契约。
  - 空文档：Markdown 文件存在但内容为空、只有标题、缺关键章节时，状态标记为待补充或阻塞。
  - 缺页面 / 缺 HTML：页面清单声明的 MVP 页面缺少对应 MD 或 HTML 时，集成契约和验收契约都必须暴露。
  - 源码引用缺失：模块、流程、验收结论缺 `code-back/mahjong` 路径时，标记未验证，不允许进入最终完成。
  - 超长路径 / 标题 / 文本：超长源码路径、模块名、流程名、风险描述不得撑破布局或截断关键信息。
  - 搜索无结果：搜索无匹配时显示推荐入口或清空筛选，不显示空白页。
  - 未知状态值：文档状态、风险等级、门禁状态出现未知值时降级为待确认，不冒充成功。
  - 重复操作：重复点击复制路径、快速切换筛选、连续跳转不应造成状态错乱。
  - 脚本失败：脚本失败、超时、路径不存在、输出为空时必须保留错误摘要并影响验收门禁。
  - 文档冲突：多人修改导致 `output/` 与 `document/` 不一致时，以 Git 差异和同步检查结果提示。
  - 同步过期：`document/` 缺失、过期或命名不一致时，不能判定交付完成。
- 明确不测：
  - 不测试真实麻将业务异常，例如断线重连、房间满员、非法出牌、匹配失败。
  - 不测试线上网络 API 超时，因为解析工作台 MVP 不依赖后端 API。
- 取舍理由：当前最真实的异常面来自本地文件、文档索引、页面渲染、脚本结果和目录同步；这些异常会直接影响解析链路可信度。

### L7 无障碍

- 确认选择：1A，测文档工作台基础无障碍。
- 测试重点：
  - 键盘可达：左侧导航、搜索框、筛选控件、复制按钮、跨页链接和 TDD 入口可通过键盘访问。
  - 焦点可见：Tab 聚焦时有清晰焦点样式，不被布局遮挡。
  - 导航语义：左侧导航和页面主区域具备清晰结构，当前页面状态可识别。
  - 搜索 / 筛选可操作：控件有可读名称，筛选结果变化可被理解。
  - 表格语义：影响范围表、AC 清单、脚本清单等表格有表头和行列关系。
  - 状态标签不只靠颜色：已验证、未验证、高风险、阻塞、可进入 TDD 等状态需要文字承载。
  - 复制按钮可读：复制源码路径、脚本路径、AC 文本等按钮有明确名称。
  - 标题层级正确：页面主标题、区域标题、卡片标题层级稳定，长文档锚点可理解。
  - 移动端可读：移动端单列阅读时文字不重叠、表格可滚动或改为可读布局。
- 明确不测：
  - 不按完整 WCAG AA 做重型合规验收。
  - 不测试复杂屏幕阅读器矩阵或多语言朗读。
- 取舍理由：当前产品是内部文档工作台，基础无障碍能保证长文档、表格、导航、筛选和复制路径这些高频操作可用。

### L8 性能基线

- 确认选择：1A，测静态工作台性能基线。
- 测试重点：
  - HTML 本地打开速度：6 个页面和设计系统展示页能在本地快速打开。
  - 首屏渲染：总览、模块地图、核心流程、影响范围、脚本索引、验收契约首屏信息可快速出现。
  - 页面切换：6 页之间跳转无明显卡顿，跨页链接可用。
  - 搜索 / 筛选响应：模块、流程、需求类型、AC、脚本等搜索筛选在常规模拟数据量下快速反馈。
  - 长表格滚动：影响范围表、AC 清单、脚本清单和测试索引长列表可滚动阅读。
  - 超长路径渲染：源码路径、脚本路径、文档路径超长时不造成布局抖动或页面横向失控。
  - 脚本扫描耗时：文件清单扫描、代码行数统计、模块索引生成、文档同步检查有基础耗时目标。
  - 同步检查耗时：`output/` 与 `document/` 对比在 MVP 规模下快速完成。
  - 无网络可读：核心内容不依赖线上服务；CDN 图标或样式不可达时，核心文档内容仍可阅读。
- 明确不测：
  - 不测试 HTTP 接口 SLA、数据库查询、服务端并发、CDN 或线上部署性能。
  - 不测试真实麻将在线对局性能。
- 取舍理由：解析工作台 MVP 是本地静态页面和脚本链路，性能基线应聚焦本地打开、文档渲染、搜索筛选、大列表、脚本扫描和无网络可读。

## 3 验收门禁

- 确认选择：1A，标准门禁。
- 完成定义：
  - `测试索引.md` 中全部 P0 + P1 用例转绿 = 可交付。
  - P2 用例尽力完成，可作为后续优化项。
  - P3 如出现，仅作为观察项或后续阶段补充。
- 覆盖率目标：
  - 核心纯逻辑：>= 90%。
  - 脚本 / 同步检查：>= 80%。
  - 页面交互：>= 70%。
- 安全 / 边界特例：
  - 安全鉴权层 P0 零失败，不可降级。
  - 边界异常层 P0 零失败，不可降级。
  - 脚本写入 `code-back/mahjong` 业务代码、P0 门禁被绕过、未验证结论冒充已验证，均直接判定未交付。
- 变异测试：
  - 当前解析 MVP 暂不启用。
  - 后续进入真实麻将前后端业务代码阶段时再评估 mutation score 门槛。
- Coding Agent 终止条件：
  - 逐条用例走红-绿-重构和回归证明。
  - 全部 P0 + P1 转绿，且覆盖率和安全/边界特例满足上述门槛，才允许标记交付。
