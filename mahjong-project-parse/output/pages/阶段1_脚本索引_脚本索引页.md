## 2. 页面概述

### 2.1 一句话定位

脚本索引页列出轻量扫描和索引脚本，明确每个脚本的用途、输入输出和安全边界。

### 2.2 存在理由

1. 脚本必须辅助文档，而不是悄悄改业务代码。
2. 没有输入输出说明，脚本结果无法被验收。
3. 没有安全边界，团队可能误用脚本。

### 2.3 页面目标（可衡量）

| 目标维度 | 具体指标 |
| :--- | :--- |
| 可理解 | 每个脚本都有用途说明 |
| 可追溯 | 每个脚本有输入输出 |
| 安全 | 每个脚本标注只读或写入范围 |

### 2.4 不做什么

- 不提供在线执行脚本能力。
- 不做复杂自动化生成。
- 不写入业务源码。

## 3. 用户与场景

### 3.1 目标用户

| 维度 | 描述 |
| :--- | :--- |
| 角色 | lei / Coding Agent |
| 技术熟练度 | 能运行本地脚本 |
| 使用设备 | 桌面浏览器和终端 |
| 使用姿势 | 看说明后在终端执行 |

### 3.2 核心使用场景

**场景 A：生成文件索引**
- 时机：源码或文档更新后。
- 前置：工作区存在目标目录。
- 动作：查看脚本说明 → 运行扫描脚本 → 回总览查看状态。
- 情绪：希望确认索引是最新的。

**场景 B：脚本失败**
- 时机：路径错误或权限不足。
- 前置：脚本执行返回错误。
- 动作：查看失败说明 → 修改路径或手工生成清单。
- 情绪：需要知道是否影响交付。

### 3.3 用户核心诉求

1. 知道脚本做什么。
2. 知道脚本不会改业务代码。
3. 知道失败后怎么办。

## 4. 信息架构

### 4.1 区域划分

1. 脚本清单。
2. 输入输出说明。
3. 安全边界。
4. 最近运行结果。
5. 故障处理。

### 4.2 区域内容

**脚本清单**
- 扫描结构。
- 统计行数。
- 生成模块索引。
- 检查文档同步。

## 5. 界面内容

### 5.1 元素清单

**脚本卡片**
- 内容：脚本名、用途、输入、输出、安全边界。
- 数据来源：脚本文档。

**运行结果**
- 内容：状态、时间、摘要。
- 数据来源：本地结果文件或人工记录。

**安全提示**
- 文案：默认只读源码，不修改业务代码。
- 显示条件：始终显示。

### 5.2 辅助信息

- 推荐命令。
- 失败原因说明。
- 验收契约入口。

## 6. 交互流程

### 6.1 页面加载与初始化

- 默认展示全部脚本。
- 标记脚本是否已创建。

### 6.2 核心操作流程

```text
查看脚本 → 确认输入输出 → 在终端运行 → 回总览或验收契约检查结果
```

### 6.3 辅助操作

- 复制脚本路径。
- 筛选只读/写索引脚本。
- 查看失败处理。

## 7. 状态与边界

### 7.1 页面状态全景

| 状态 | 触发条件 | 页面表现 |
| :--- | :--- | :--- |
| 初始 | 脚本清单存在 | 展示脚本卡片 |
| 输入 | 筛选脚本 | 更新列表 |
| 成功 | 最近运行成功 | 展示摘要 |
| 失败 | 最近运行失败 | 展示错误 |
| 异常 | 脚本越界 | 标记不可用 |
| 空状态 | 尚未添加脚本 | 提示第一版脚本边界 |

### 7.2 边界与容错

- 网络异常：无需网络。
- 会话过期：无登录态。
- 连续错误：保留最后错误。
- 异常输入：路径不存在时提示。
- 并发操作：多人改脚本时以 Git 差异为准。

## 8. 业务规则

- 默认不执行脚本，只展示说明。
- 脚本如会写文件，必须只写索引或文档产物。
- 涉及业务代码写入的脚本不得进入第一版。

## 11. 验收标准

### 功能验收

- 至少展示 4 类脚本用途。
- 每个脚本有输入输出。
- 每个脚本有安全边界。
- 能复制脚本路径。
- 能跳转验收契约。

### 体验验收

- 只读边界醒目。
- 失败说明清晰。
- 脚本状态可扫描。
- 命令文案可复制。
- 移动端可读。

### 安全验收

- 不提供在线写源码按钮。
- 越界脚本标记不可用。
- 不隐藏失败记录。
- 不展示敏感路径之外的信息。
- Git 差异作为最终确认。
