## 2. 页面概述

### 2.1 一句话定位

总览页是项目解析工作台的第一屏，用于让 lei 和 he 快速知道源码解析做到了哪里、还有什么风险、下一步应该看哪份文档。

### 2.2 存在理由

1. 没有总览页，用户需要在多个文档之间猜测当前进度。
2. 没有风险聚合，未验证结论容易被当成确定结论使用。
3. 没有下一步入口，design-master 和 tdd-master 的链路承接不清晰。

### 2.3 页面目标（可衡量）

| 目标维度 | 具体指标 |
| :--- | :--- |
| 入口效率 | 用户 10 秒内能找到模块地图、核心流程或验收契约入口 |
| 风险可见 | 高风险和未验证项在首屏可见 |
| 状态清晰 | 文档完成度、源码引用覆盖、验收门禁状态清晰展示 |

### 2.4 不做什么

- 不做登录入口，因为第一版所有人看到同一套内容。
- 不做复杂编辑表单，因为文档更新仍以 Markdown 为准。
- 不做业务麻将玩法操作，因为第一版不改业务代码。

## 3. 用户与场景

### 3.1 目标用户

| 维度 | 描述 |
| :--- | :--- |
| 角色 | lei / he / Coding Agent |
| 技术熟练度 | 中等到较高 |
| 使用设备 | 桌面浏览器为主 |
| 使用姿势 | 对照文档、源码和验收项进行走查 |

### 3.2 核心使用场景

**场景 A：开始一次项目走查**
- 时机：打开项目解析工作台。
- 前置：PRD 和设计蓝图已生成。
- 动作：查看完成度 → 点击模块地图或核心流程。
- 情绪：需要快速建立方向感。

**场景 B：发现验收阻塞**
- 时机：准备进入 TDD 前。
- 前置：部分文档缺源码引用。
- 动作：查看风险卡片 → 跳转验收契约页。
- 情绪：希望明确缺口，不想凭感觉推进。

### 3.3 用户核心诉求

1. 快速知道当前状态。
2. 找到下一步入口。
3. 看到风险和未验证项。

## 4. 信息架构

### 4.1 区域划分

1. 左侧导航。
2. 顶部标题和全局搜索。
3. 状态统计区。
4. Bento 信息卡片区。
5. 最近文档和下一步区。

### 4.2 区域内容

**状态统计区**
- 文档完成度。
- 源码引用覆盖。
- 验收门禁完成度。
- 高风险项数量。

**Bento 信息卡片区**
- 项目画像摘要。
- 模块地图入口。
- 核心流程入口。
- 影响范围入口。
- 验收契约入口。

## 5. 界面内容

### 5.1 元素清单

**全局搜索**
- 占位符：“搜索模块、流程、源码引用、验收项”
- 数据来源：本地 JSON 索引。
- 显示条件：始终显示。

**状态卡片**
- 内容：完成度、风险、未验证项。
- 数据来源：文档索引和验收契约。
- 显示条件：始终显示。

**下一步按钮**
- 文案：“查看验收契约”“进入模块地图”。
- 显示条件：始终显示。

### 5.2 辅助信息

- 最近更新时间。
- 数据来源说明。
- 当前阶段标记：阶段1(MVP)。

## 6. 交互流程

### 6.1 页面加载与初始化

- 读取本地索引。
- 展示完成度和最近文档。
- 如果索引缺失，显示错误状态并引导到脚本索引页。

### 6.2 核心操作流程

```text
打开总览页 → 查看状态卡片 → 选择下一步入口
  ├─ 需要理解结构 → 模块地图页
  ├─ 需要理解流程 → 核心流程页
  └─ 准备验收 → 验收契约页
```

### 6.3 辅助操作

- 搜索关键字。
- 按角色视角筛选推荐阅读。
- 复制源码路径。

## 7. 状态与边界

### 7.1 页面状态全景

| 状态 | 触发条件 | 页面表现 |
| :--- | :--- | :--- |
| 初始 | 成功读取索引 | 展示完整工作台 |
| 加载 | 正在读取本地索引 | 展示骨架屏 |
| 空状态 | 尚未生成项目画像 | 提示查看 PRD 或运行扫描脚本 |
| 失败 | 索引文件缺失 | 显示错误块并跳转脚本索引 |
| 异常 | 文档状态不一致 | 显示待处理风险 |
| 成功 | 门禁齐全 | 显示可进入 TDD |

### 7.2 边界与容错

- 网络异常：不依赖网络，提示“本地文件可继续查看”。
- 会话过期：无登录态，不适用。
- 连续错误：保留错误摘要，避免重复弹窗。
- 异常输入：搜索无结果时显示推荐入口。
- 并发操作：多人改文档时提示以 Git 差异为准。

## 8. 业务规则

- 总览页不产生业务数据，只聚合文档和索引状态。
- 未验证结论必须显式展示。
- 所有入口都指向已生成页面或明确标记为后续阶段。

## 9. 页面流转

  ├─ 点击模块地图 → 模块地图页
  ├─ 点击核心流程 → 核心流程页
  ├─ 点击影响范围 → 影响范围页
  ├─ 点击脚本索引 → 脚本索引页

## 11. 验收标准

### 功能验收

- 能展示 4 个以上状态指标。
- 能跳转到 5 个核心页面。
- 能显示最近文档。
- 能展示高风险项。
- 能搜索模块或流程。

### 体验验收

- 首屏能看懂项目当前状态。
- 状态标签含义明确。
- 入口按钮不混淆。
- 移动端可读。
- 长文本不挤压布局。

### 安全验收

- 不展示真实用户隐私。
- 不提供业务代码修改入口。
- 脚本入口说明只读边界。
- 缺源码引用时必须提示。
- 断链必须可发现。
