## 1. 文档元信息

| 字段 | 内容 |
| :--- | :--- |

| 下游去向 | 影响范围页 / 验收契约页 |

## 2. 页面概述

### 2.1 一句话定位

核心流程页把麻将后端的房间、用户、牌局、匹配、RPC 和异常恢复流程整理成可复述的路径。

### 2.2 存在理由

1. 只有模块名不能说明系统如何运转。
2. 新需求通常影响流程，而不仅是单个文件。
3. 没有异常路径，验收契约会漏掉风险。

### 2.3 页面目标（可衡量）

| 目标维度 | 具体指标 |
| :--- | :--- |
| 流程完整 | 覆盖 6 类核心流程 |
| 可追溯 | 每条流程有涉及模块和源码引用 |
| 可验收 | 每条流程能对应验收点 |

### 2.4 不做什么

- 不实现麻将规则。
- 不画所有细枝末节。
- 不替代源码阅读。

## 3. 用户与场景

### 3.1 目标用户

| 维度 | 描述 |
| :--- | :--- |
| 角色 | lei / he |
| 技术熟练度 | 能理解流程和状态 |
| 使用设备 | 桌面浏览器 |
| 使用姿势 | 按流程对照模块地图 |

### 3.2 核心使用场景

**场景 A：讲清房间生命周期**
- 时机：新人理解项目时。
- 前置：模块地图已读。
- 动作：选择房间流程 → 查看步骤和异常路径。
- 情绪：需要清楚主线和边界。

**场景 B：评估新增玩法**
- 时机：he 提出规则变化。
- 前置：知道规则涉及牌局流程。
- 动作：查看牌局流程 → 跳转影响范围页。
- 情绪：希望知道影响模块。

### 3.3 用户核心诉求

1. 明确流程主线。
2. 看见异常路径。
3. 找到影响模块。

## 4. 信息架构

### 4.1 区域划分

1. 流程目录。
2. 流程图。
3. 步骤说明。
4. 涉及模块。
5. 验收点。

### 4.2 区域内容

**流程目录**
- 房间生命周期。
- 用户状态。
- 牌局运行。
- 匹配玩法。
- RPC 外部边界。
- 异常恢复。

## 5. 界面内容

### 5.1 元素清单

**流程选择器**
- 类型：分段控件。
- 默认：房间生命周期。
- 数据来源：核心流程索引。

**流程图**
- 内容：步骤、分支、异常路径。
- 数据来源：Markdown 流程文档。

**源码引用**
- 内容：涉及文件。
- 操作：复制路径。

### 5.2 辅助信息

- 验证状态。
- 影响需求类型。
- 下一步建议。

## 6. 交互流程

### 6.1 页面加载与初始化

- 默认展示房间生命周期。
- 右侧显示本流程涉及模块。

### 6.2 核心操作流程

```text
选择流程 → 查看流程图 → 查看涉及模块 → 查看验收点 → 跳转影响范围
```

### 6.3 辅助操作

- 按状态筛选已验证/未验证。
- 复制源码路径。
- 跳转验收契约。

## 7. 状态与边界

### 7.1 页面状态全景

| 状态 | 触发条件 | 页面表现 |
| :--- | :--- | :--- |
| 初始 | 成功读取流程索引 | 展示默认流程 |
| 输入 | 切换流程 | 更新流程图 |
| 成功 | 流程有引用 | 展示已验证 |
| 失败 | 缺少源码引用 | 标记未验证 |
| 异常 | 流程解析失败 | 显示错误 |
| 空状态 | 无流程文档 | 引导回模块地图 |

### 7.2 边界与容错

- 网络异常：无需网络。
- 会话过期：无登录态。
- 连续错误：流程错误合并展示。
- 异常输入：未知流程回退默认流程。
- 并发操作：多人改文档时提示以 Git 差异合并。

## 8. 业务规则

- 每条核心流程必须包含涉及模块。
- 异常路径不得省略。
- 未验证流程不能进入最终验收。

## 9. 页面流转

```text
核心流程页
  ├─ 点击影响范围 → 影响范围页
  └─ 点击验收契约 → 验收契约页
```

| :--- | :--- | :--- |
| 流程列表 | Markdown / JSON | 流程导航 |

## 11. 验收标准

### 功能验收

- 覆盖至少 6 类流程。
- 每条流程展示涉及模块。
- 每条流程有验收点。
- 可跳转影响范围页。
- 可跳转验收契约页。

### 体验验收

- 流程图清晰。
- 分支和异常可见。
- 源码路径易复制。
- 状态标签不混淆。
- 移动端可阅读。

### 安全验收

- 不复制大段源码。
- 未验证流程有标记。
- 不提供源码修改入口。
- 错误状态不泄露敏感数据。
- 文档冲突可发现。
