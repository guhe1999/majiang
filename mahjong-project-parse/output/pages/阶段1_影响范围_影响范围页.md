# 页面名称：影响范围页

| 字段 | 内容 |
| :--- | :--- |

## 2. 页面概述

### 2.1 一句话定位

影响范围页帮助 he 把新需求映射到模块、流程、风险和验收点。

### 2.2 存在理由

1. 新需求如果没有影响判断，容易直接改错模块。
2. he 不需要逐行读源码，但需要知道该找哪里。
3. 验收点需要从影响范围中自然产生。

### 2.3 页面目标（可衡量）

| 目标维度 | 具体指标 |
| :--- | :--- |
| 需求归类 | 至少覆盖 6 类常见需求 |
| 影响判断 | 每类需求有模块和流程映射 |
| 验收衔接 | 每类需求有推荐验收点 |

### 2.4 不做什么

- 不做需求提交系统。
- 不自动生成代码任务。
- 不替代人工评审。

## 3. 用户与场景

### 3.1 目标用户

| 维度 | 描述 |
| :--- | :--- |
| 角色 | he 为主，lei 辅助确认 |
| 技术熟练度 | 不要求熟悉全部源码 |
| 使用设备 | 桌面浏览器 |
| 使用姿势 | 提需求前快速查表 |

### 3.2 核心使用场景

**场景 A：新增玩法规则**
- 时机：he 提出规则调整。
- 前置：知道需求类型。
- 动作：筛选“玩法规则” → 查看影响模块和流程。
- 情绪：希望知道是否高风险。

**场景 B：需求无法归类**
- 时机：需求描述很新。
- 前置：表中没有匹配类别。
- 动作：标记开放问题 → 回核心流程页确认。
- 情绪：不确定但需要有下一步。

### 3.3 用户核心诉求

1. 快速归类需求。
2. 看到风险等级。
3. 找到推荐阅读和验收点。

## 4. 信息架构

### 4.1 区域划分

1. 需求类型筛选。
2. 影响范围表。
3. 风险摘要。
4. 推荐阅读。
5. 验收点。

### 4.2 区域内容

**影响范围表**
- 需求类型。
- 影响模块。
- 影响流程。
- 风险等级。
- 推荐验收点。

## 5. 界面内容

### 5.1 元素清单

**需求类型筛选**
- 选项：玩法规则、房间流程、用户行为、匹配策略、日志记录、异常恢复。
- 数据来源：影响范围矩阵。

**风险标签**
- 选项：高风险、中风险、低风险。
- 显示条件：每条需求类型。

**推荐阅读**
- 内容：模块地图或核心流程链接。
- 显示条件：每条影响范围。

### 5.2 辅助信息

- 使用说明。
- 未归类需求提示。
- 验收契约入口。

## 6. 交互流程

### 6.1 页面加载与初始化

- 默认展示全部需求类型。
- 高风险项置顶。

### 6.2 核心操作流程

```text
选择需求类型 → 查看影响模块和流程 → 查看风险等级 → 跳转推荐阅读 → 补充验收点
```

### 6.3 辅助操作

- 搜索需求关键词。
- 按风险等级筛选。
- 复制影响范围描述。

## 7. 状态与边界

### 7.1 页面状态全景

| 状态 | 触发条件 | 页面表现 |
| :--- | :--- | :--- |
| 初始 | 影响矩阵存在 | 展示需求类型表 |
| 输入 | 搜索或筛选 | 更新表格 |
| 成功 | 找到匹配类型 | 展示影响范围 |
| 失败 | 无匹配类型 | 提示标记开放问题 |
| 异常 | 数据缺失 | 引导验收契约 |
| 空状态 | 尚无需求样例 | 展示 PRD 模拟需求 |

### 7.2 边界与容错

- 网络异常：无需网络。
- 会话过期：无登录态。
- 连续错误：保留筛选条件。
- 异常输入：空搜索回到全部列表。
- 并发操作：文档冲突以 Git 差异解决。

## 8. 业务规则

- 高风险需求必须推荐先读核心流程。
- 无法归类的需求不得直接进入实现。
- 影响范围不是代码修改清单，只是评审入口。

## 9. 页面流转

```text
影响范围页
  ├─ 点击核心流程 → 核心流程页
  └─ 点击验收契约 → 验收契约页
```

## 10. 数据需求

| 数据 | 来源 | 用途 |
| :--- | :--- | :--- |
| 需求类型 | 影响矩阵 | 表格行 |
| 风险等级 | 人工解析 | 排序和标签 |
| 验收点 | PRD / 验收契约 | 门禁衔接 |

## 11. 验收标准

### 功能验收

- 至少 6 类需求类型。
- 每类有模块映射。
- 每类有流程映射。
- 每类有风险等级。
- 每类有验收建议。

### 体验验收

- 高风险项醒目。
- 搜索和筛选可用。
- 推荐阅读明确。
- 无结果有下一步。
- 移动端表格可读。

### 安全验收

- 不自动生成代码修改。
- 不隐藏高风险。
- 无法归类要提示。
- 不展示敏感数据。
- 不提供业务代码写入入口。
