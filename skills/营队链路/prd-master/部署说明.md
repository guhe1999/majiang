# PRD 大师 · 部署说明

## 1. 文件清单

```
prd-master/
├── SKILL.md              ← 主 Controller（必须）
├── README.md             ← 项目说明
├── docs/ARCHITECTURE.md  ← 架构决策
├── agents/               ← 5 个子 Agent prompt
├── playbooks/            ← 三品类调研打法
├── templates/            ← 交付物模板
└── validators/           ← Python 硬校验脚本
```

## 2. Claude Code 部署（推荐）

### 步骤 1：拷贝到全局 skills 目录

**Mac / Linux**：
```bash
mkdir -p ~/.claude/skills/prd-master
cp -r prd-master/* ~/.claude/skills/prd-master/
```

**Windows (PowerShell)**：
```powershell
New-Item -ItemType Directory -Path "$HOME\.claude\skills\prd-master" -Force
Copy-Item -Path "prd-master\*" -Destination "$HOME\.claude\skills\prd-master\" -Recurse -Force
```

### 步骤 2：验证 Python 可用

```bash
py --version   # Windows
python3 --version   # Mac/Linux
```

需要 3.10+。

### 步骤 3：启动 Claude Code，触发 Skill

```
> 启动 PRD 大师
> 或：用 prd-master 帮我写 PRD
```

Claude Code 会自动加载 `~/.claude/skills/prd-master/SKILL.md` 作为指令。

## 3. Codex / Cursor / 其他 AI Coding 工具部署

把 `SKILL.md` 内容贴成 system prompt（或项目级 .codex / .cursorrules）。

子 Agent (`agents/*.md`) 在调用时按需附加。

## 4. 项目目录约定

PRD 大师会把每个 PRD 落地在：

```
{your-workspace}/projects/{项目名-短描述}/
├── PRD.md
├── assumptions.md
├── debate-log.md
├── conversation.md
└── evidence/
    ├── competitors.md
    └── benchmark.md
```

建议在你工作区建一个 `projects/` 文件夹专门放产出。

## 5. 跨项目证据库（可选）

PRD 大师调研得到的竞品/Benchmark 可以缓存到：

```
{your-workspace}/evidence-cache/
└── {品类}/
    └── {产品名}/
        ├── competitors.md
        └── benchmark.md
```

下次同领域 PRD 可以复用，不重复抓。

## 6. 常见部署问题

### Q: Claude Code 找不到 skill

确认：
- `~/.claude/skills/prd-master/SKILL.md` 存在
- SKILL.md 的 frontmatter 里 `name: prd-master` 没改
- Claude Code 已重启或新会话

### Q: Python 脚本编码错误（Windows）

Windows 中文系统默认 GBK，跑脚本前设置：

```powershell
$env:PYTHONIOENCODING="utf-8"
$env:PYTHONUTF8="1"
```

或直接：

```powershell
py -X utf8 prd-master\validators\check_evidence.py path\to\PRD.md
```

### Q: WebSearch / WebFetch 失败

- 检查你的 Claude Code 是否在美国可用区（WebSearch 限制）
- 备选：用 Bash 直接 curl + py 解析
- 国内场景调研可以用 gh CLI 替代

### Q: 子 Agent 调用慢

- 4 个 Reviewer 并行调用比串行快很多
- 在主 SKILL.md 第 3.2 节，确保 4 个 Agent 在同一消息里调用（并行）

## 7. 升级路径

PRD 大师后续会迭代。升级时：

```bash
# 进入你的 prd-master 副本
git pull   # 如果你 fork 了仓库
# 或者重新拷贝
cp -r prd-master/* ~/.claude/skills/prd-master/
```

注意不要覆盖你自己改过的 prompt（如本地化的）。
