# 移动 App 调研 Playbook

## 适用场景

C 端 iOS/Android App、小程序、Web App（C 端）、消费类产品

## 一、找竞品（5-8 个）

### 直接竞品（3-5 个）

**找法**：
- AppStore 搜索关键词 → 看 TOP 10 排名
- 七麦数据 / App Annie / Sensor Tower 看品类榜
- WebSearch：`{产品定位关键词} App 推荐`
- 小红书搜 `{品类} App 推荐` / `{品类} App 对比`
- 知乎搜 `好用的 {品类} App`

**记录**：
- App 名 / 公司 / iOS+Android 都有吗
- 一句话定位
- 核心功能（3-5 个）
- 商业模式（订阅/广告/电商/打赏/付费下载）
- 公开下载量 / DAU / MAU（找得到的）
- 主要用户画像

### 间接竞品（2-3 个）

竞争用户时间/钱包的产品。例：做"健身打卡 App" → 间接竞品可能是 Keep（已是垄断）、微信运动（免费且天然 viral）、传统线下健身房

## 二、抓 App Store 页面

**用 WebFetch 抓**：
- AppStore 详情页（提取截图描述、版本更新历史、评论）
- Google Play 详情页
- 官网（如果有）

**关键字段**：
- AppStore 描述（首句最重要）
- 截图数量 + 各截图标题（暴露作者主推什么）
- 最新版本更新内容（暴露当前迭代方向）
- 评分分布（5/4/3/2/1 星各多少）
- 总评论数

## 三、挖差评（C 端核心信号源）

**C 端的差评是最便宜的用研**。

### 必查渠道

| 渠道 | 用法 |
|------|------|
| **AppStore** | 详情页 → 评论 → 排序按"最新" + 筛选"1星 / 2星" |
| **Google Play** | 同 AppStore |
| **小红书** | 搜 `{App名} 缺点` / `{App名} 卸载` / `{App名} 替代` |
| **知乎** | 搜 `{App名} 为什么不好用` / `{App名} 替代` |
| **微博** | 搜 `{App名} 难用` / `{App名} bug` |
| **B 站** | 搜 `{App名} 测评` 看负面测评视频的评论区 |

### 挖什么

- **卸载原因 Top 10**
- **功能缺失抱怨 Top 5**
- **性能/卡顿/闪退抱怨 Top 5**
- **广告/付费抱怨 Top 5**
- **客服/退款抱怨 Top 5**
- **隐私/数据担忧 Top 5**

### 输出格式

每条原话引用 + 来源 + 日期。

## 四、行业 Benchmark

### 必查指标

| 指标 | 健康范围 | 数据源 |
|------|---------|--------|
| **次日留存** | >30% (健康), >40% (优秀) | Sensor Tower, AppsFlyer benchmarks |
| **7 日留存** | >15% (健康), >25% (优秀) | 同上 |
| **30 日留存** | >5% (健康), >10% (优秀) | 同上 |
| **DAU/MAU 黏性** | >20% (健康), >50% (高黏性) | 同上 |
| **平均使用时长** | 因品类大相径庭 | QuestMobile, Mob 研究院 |
| **付费转化率** | 免费下载 → 付费 1-3% (健康), >5% (优秀) | RevenueCat State of Subscription |
| **ARPU/ARPPU** | 因品类大相径庭 | App Annie, Sensor Tower |
| **CAC** | 注意安卓和 iOS 差异巨大 | Liftoff, AppsFlyer |
| **AppStore 评分** | >4.5 (健康) | 直接看 |

### 数据源

- **Sensor Tower**（App 下载/营收数据）
- **App Annie / data.ai**（同上）
- **QuestMobile**（中国市场，DAU/MAU 等）
- **Mob 研究院**（中国市场报告）
- **极光数据**（中国市场）
- **AppsFlyer Benchmarks**（归因、留存）
- **RevenueCat State of Subscription**（订阅类）
- **Liftoff Mobile App Trends**（CAC）

## 五、关键判断辅助

### 这个 App 值不值得做的检查清单

- [ ] 是否有高频使用场景（每周>3 次理想，每天最佳）
- [ ] 是否有自然传播触发点（分享/邀请/拼团/榜单）
- [ ] 冷启动方案是否清晰（应用市场 ASO / 内容 / KOC / 合作 / 投放）
- [ ] 留存机制是否设计（消息推送 / 内容更新 / 任务体系 / 社交关系）
- [ ] 变现模型是否清晰（订阅 / 内购 / 广告 / 电商）
- [ ] 监管风险（涉黄涉政 / 数据出境 / 未成年保护 / 平台抽成）

### 中国 C 端 App 特殊点

- 平台分发：AppStore 中国区 + 安卓 8 大应用市场（华米OV+腾讯+百度+360 等）
- ASO（应用商店优化）极其重要
- iOS 中国区订阅付费转化率约为美国的 1/3
- 短视频/直播带货是新获客主战场（抖音/快手/视频号）
- 用户对广告极度反感，付费意愿低
- 隐私合规：个保法 + 数据安全法 + 等保
