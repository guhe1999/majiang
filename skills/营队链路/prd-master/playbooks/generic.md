# 通用调研 Playbook（v0.3 P2-8 · 冷门品类 fallback）

## 适用场景

不在 saas / mobile-app / ecommerce 三大主品类的项目：
- 硬件 / IoT / 智能设备
- 医疗 / 生物科技
- 金融 / 保险 / 资管
- 教育 / 培训 / 知识付费
- 制造 / 工业 / 供应链
- 政企服务 / 公共领域
- 内容创作工具
- AI 模型 / 算法服务
- 任何混合品类（如 SaaS + 硬件）

## 总原则

通用 playbook 牺牲深度换覆盖度。能力比 3 大主 playbook 弱 30%，但能接住冷门品类。

具体项目用了通用 playbook 后，建议跑完一次后**生成项目专属的 playbook 补丁**（标注"建议补充到 playbooks/{品类}.md"），下次同类项目就有更深的打法。

## 一、找竞品（5-8 个）

### 找法（通用）

按相关性顺序：

1. **WebSearch 关键词组合**：
   - `{产品定位} alternatives`
   - `best {品类} {2026}`
   - `{知名头部产品} vs / competitors`
   - `{细分场景} solutions`
2. **垂直行业目录**：
   - 硬件：CES 获奖产品、Indiegogo/Kickstarter top
   - 医疗：FDA 注册产品、Pitchbook 医疗赛道
   - 金融：Fintech100、巴塞尔合规产品
   - 教育：得到/混沌/樊登/三节课等课程平台
   - 制造：阿里 1688、慧聪、行业展会获奖
3. **行业垂直媒体**：
   - 36 氪 / 虎嗅 / TechCrunch 按品类筛选近 1 年报道
4. **学术圈**：
   - 如果是技术驱动产品（AI 模型、生物科技等），arxiv + Google Scholar 找 SOTA 论文
   - Demo 站、开源 repo 也算"竞品"

### 记录

- 公司/产品 / 一句话定位
- 核心功能 3-5 个
- 商业模式（订阅 / 一次性 / 服务费 / 硬件 + 服务 / 混合）
- 用户规模或营收（找得到的标 A 级，找不到的诚实标"未公开"）

## 二、抓官网/产品页

WebFetch 抓：
- 主页（提取定位 + 价值主张）
- 产品/功能页（提取功能列表）
- 价格页（如有）
- 关于我们/团队（提取背景）

冷门品类特殊点：
- 硬件：抓产品规格表、官方测评、京东/天猫产品页
- 医疗：抓 NMPA 注册信息、FDA 510(k)、临床试验数据库
- 金融：抓产品说明书、风险评级、监管备案

## 三、挖差评（用户痛点的代用品）

### 通用渠道

| 渠道 | 通用性 | 覆盖品类 |
|------|--------|---------|
| 知乎"为什么放弃 XX" / "XX 缺点" | 高 | 全部 |
| 小红书"XX 测评" / "XX 踩雷" | 高 | C 端为主 |
| 微博搜品牌名 | 中 | 全部 |
| 黑猫投诉 tousu.sina.com.cn | 高 | C 端、有客服的 B 端 |
| Reddit 子版块（r/{品类}） | 中 | 海外为主 |
| 行业垂直论坛 | 高 | 因品类而异（如硬件 = Mobile01、医疗 = 丁香园）|

### 冷门品类专属

| 品类 | 挖差评的"非常规"地方 |
|------|--------------------|
| 硬件 | 京东问大家、贴吧、什么值得买 |
| 医疗 | 丁香园、医脉通、专业医生群 |
| 金融 | 雪球评论、东方财富股吧、私募排排网 |
| 教育 | 黑猫投诉、行业自媒体（如"芥末堆"）|
| 制造 | 阿里 1688 买家评论、行业展会反馈 |
| 政企 | 招投标平台中标后评估、政府采购年度评价 |
| AI 模型 | HuggingFace 评论、arxiv 引用 + 批评论文 |

### 挖什么

- 退货/取消原因 Top 5
- 性能不达预期 Top 5
- 体验差点 Top 5
- 客服/售后差 Top 5
- 性价比抱怨 Top 5
- 合规/安全担忧 Top 5

## 四、行业 Benchmark

### 通用数据源

- **艾瑞咨询**（全品类行业报告）
- **易观分析**
- **国家统计局**专题数据
- **Statista**（全球）
- **eMarketer**
- **Pitchbook / Crunchbase**（融资数据）

### 冷门品类专属

| 品类 | 报告来源 |
|------|---------|
| 硬件 | IDC / Counterpoint / Strategy Analytics |
| 医疗 | EvaluatePharma / Frost & Sullivan / WHO / NMPA |
| 金融 | 巴塞尔委员会 / 中国人民银行 / Wind / Bloomberg |
| 教育 | iiMedia / 多鲸资本 / 教育部 |
| 制造 | 中国信通院 / 工信部 / IEEE / Gartner |
| 政企 | 政府公开年报 / 行业协会报告 |
| AI 模型 | OpenAI / Anthropic / DeepMind 公开评测、HuggingFace Leaderboard |

### 找不到 benchmark 时

- 老老实实标"未找到公开资料"
- 给出"如何自己测/采集 benchmark 的方法"
- 进 assumptions.md，上线后用真实数据校准

## 五、品类混合项目处理

### 情况 A: 主品类清晰、附带次品类

例：主品类是 SaaS，但附带一个手机端 App 模块。

**做法**：
- 主用 saas.md playbook
- App 部分参考 mobile-app.md 的差评挖掘和 benchmark
- 在 evidence/ 加一个 `mobile-app-addendum.md` 子文档

### 情况 B: 两个品类同等重要

例：硬件 + 配套 App + SaaS 后台（典型 IoT 项目）。

**做法**：
- 用本通用 playbook 作为主框架
- 每个品类各跑一遍核心调研
- 在 evidence/ 分目录：`hardware/`, `app/`, `saas/`

### 情况 C: 完全新品类（罕见）

例：太空旅游、脑机接口等。

**做法**：
- 用通用 playbook，调研范围限定到"最接近的已有市场"
- 大量标"未找到公开资料"+"待早期 PMF 验证"
- 进 assumptions.md 标 D-E 级别 + 高风险
- 战略评审 R1 重点质疑"是否该现在做"

## 六、生成"品类专属 playbook 补丁"

跑完一次冷门品类项目后，Lead PM **主动**生成一份补丁：

```markdown
# {品类} playbook 补丁建议

跑了本项目后发现以下知识应该沉淀到正式 playbook：

## 这次有用的竞品来源
- ...

## 这次有用的差评渠道
- ...

## 这次有用的 benchmark 数据源
- ...

## 这次发现的品类特殊点
- ...

## 建议下次同类项目用的 playbook 章节顺序
- ...
```

写到 `playbooks/contributions/{品类}-{日期}.md`，未来正式品类 playbook 可参考。

## 七、给企业家的"调研深度档位"（v0.3 新加）

通用 playbook 默认走"标准档"。但有的项目企业家会要求快/深，提供 3 档：

| 档位 | 时间 | 竞品数 | 深度 | 适合场景 |
|------|------|--------|------|---------|
| 🚀 快速 | 3-5 分钟 | 3 个直接 | 仅核心定位+定价 | 初步 idea、内部讨论 |
| 🎯 标准 | 5-15 分钟 | 5-8 个混合 | + 差评挖掘 + benchmark | 默认（推荐） |
| 🔬 深度 | 20-40 分钟 | 10-15 个 | + 用户访谈代用 + 学术文献 | 重大决策、外部融资 |

阶段 2 启动时用 AskUserQuestion 让企业家选档位。
