**项目名**: fba-smart-restock
**产品名（暂定）**: FBA 智能补货决策助手（内部代号）
**版本**: 锚点 V2（基于用户修正：每日决策 + MVP 虚拟数据集）
**日期**: 2026-05-23

---

## 1. 一句话定位

每天给我一份"哪些 SKU 该补、走海运还是空运"的建议清单，平时缺货风险随时预警。

## 1.5 MVP 数据策略（v1 关键决策）

**v1 用虚拟数据集（mock data）跑通算法和决策引擎，真实数据接入推迟到 v1.5。**

理由：
- 真实数据接入是工程量大、不确定性高的环节（特别是海运/空运报价 API）
- 算法和决策逻辑可以用模拟数据集做回测验证可行性
- 验证完算法值得做后再投资真实数据接入

具体含义：
- v1 提供"虚拟数据生成器"：可配置销量分布、库存初值、广告投放波动、物流时效/价格波动
- 算法和决策引擎在虚拟数据上跑回测，证明误差和决策质量
- v1.5 添加真实数据接入层（FBA / 销量 / 报价 API），其他逻辑不变

## 3. 场景

### 3.1 主场景：每天早上看补货日报

- 凌晨系统跑预测和决策计算
- 早上 9 点你打开看日报：
  - **今日建议发货清单**：SKU × 数量 × 海运 / 空运 / 不发 × 推荐理由
  - **库存全景**：每个 SKU 当前库存、预计可卖天数、波动趋势
  - **预警面板**：哪些 SKU 60 天内可能断货，提示让工厂尽快下单
- 你拍板调整后下单（或交助理执行）

**注**：每日决策不代表每日都发货——大多数日子建议清单可能是空的（库存充足）或只有 1-2 个 SKU。频率高是为了"及时发现"，不是"频繁发货"。

### 3.2 副场景：缺货预警随时推

- 任何 SKU 库存跌破"安全天数"阈值 → 立刻推送
- 销量突然异常上涨（比如广告投放生效）→ 立刻预警
- 物流方报价大幅变化 → 推送（影响下次决策成本）

### 3.3 当前替代方案及问题

- **现状**: Excel 手工算 + 凭直觉判断
- **问题**:
  - SKU 多了 200 个之后，每天/每周看一遍 Excel 不现实
  - 凭直觉常出现两类错误：保守过头（资金积压）、激进过头（缺货）
  - 海空运成本对比靠记忆，没有实时数据
  - 销量波动（广告/排名变化）没有系统化捕捉，凭印象

## 4. 核心痛点

### 4.1 真正的痛（按损失大小排序）

1. 🔴 **缺货损失**（核心痛点）
   - 直接损失：销售额 = 日销 × 缺货天数 × 客单价
   - 隐性损失：Listing 排名下降 → 恢复销量需要 2-4 周 + 广告投放代偿
   - 经验：缺货损失 ≈ 3-5 倍直接销售额
2. 🟠 **资金占用**
   - 200 SKU 备货积压资金（数十万到上百万级）
   - 资金机会成本：年化 8-15%
3. 🟠 **FBA 仓储费**
   - 长期仓储费（>365 天） + 月度仓储费
   - 滞销 SKU 月费 + 长期费可能反超商品价值
4. 🟡 **海运/空运决策错误**
   - 该走海运的走空运 → 物流成本 5-10 倍
   - 该走空运的走海运 → 错过补货窗口 → 缺货损失

### 4.2 痛点等级总判断

🔴 **致命** —— 当前 Excel + 直觉模式注定会犯错，且 SKU 越多错得越频繁

## 6. 功能范围

### 6.1 必做（v1 核心闭环，全部基于虚拟数据集）

- **虚拟数据生成器**：可配置参数模拟 FBA 库存、销量、广告投放、Listing 排名、海运/空运时效和价格波动
- **销量预测模型**（特征：销量历史 + 广告投放 + Listing 排名 + 库存状态）
- **补货决策引擎**：基于成本目标函数（缺货损失 vs 资金占用 vs 仓储费 vs 物流成本）
- **海空运选择算法**：基于剩余安全库存天数 vs 海运/空运时效和价差
- **每日补货日报**（清单 + 推荐理由 + 决策置信度）
- **缺货预警**（库存阈值跌破 / 销量异常上升 → 推送）
- **决策可解释性面板**：每条建议背后的关键数据 + 假设 + 替代方案
- **算法回测能力**：在虚拟数据集上跑过去 N 天的"如果按系统建议执行"vs"按当前策略执行"对比

### 6.2 应做（v1 含）

- 库存全景仪表盘（每个 SKU 当前状态）
- 上游下单预警（60/90 天后断货风险）
- SKU 自动分类（稳定 / 波动 / 季节性 / 长尾）→ 不同预测策略

### 6.3 可做（v1.5 后）

- **真实数据接入层**（FBA / 亚马逊销量 / 海运/空运报价 API）—— v1 验证算法值得做后再投入
- 滞销 SKU 清仓建议
- 季节性活动预测（黑五/Prime Day/圣诞）特殊模型
- A/B 测试不同决策策略
- SaaS 化（多租户支持）

### 6.4 不做（边界外）

- ❌ 工厂下单/采购管理
- ❌ Listing 优化建议
- ❌ 广告投放优化
- ❌ 实际下单海/空运（只给建议，你或助理执行）
- ❌ 财务对账

## 9. 最大不确定性

1. 🔴 **销量预测模型准确度**：不稳定的 SKU 预测准吗？特别是受广告影响的
2. 🔴 **海运/空运报价 API**：实时报价能拿到吗？还是要手工维护？
3. 🟠 **缺货损失量化模型**：到底用什么公式估算"缺货 1 天的损失"？需要历史数据校准
4. 🟠 **决策建议接受率**：你会不会觉得系统"不懂行"经常推翻？
5. 🟡 **响应延迟**：预警从触发到推送，1 小时内能做到吗？

每条都进了 assumptions.md，会逐条设计验证方案。
