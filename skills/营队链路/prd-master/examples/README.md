# 案例库 · PRD 大师产出实例

每个案例是一次完整的 PRD 大师跑通过程的存档。包含场景锚点、调研、辩论、PRD 终稿、假设清单等全套交付物。

## 当前案例

### 01-fba-smart-restock · FBA 智能补货决策引擎

**企业家**: 刘润（润米咨询）
**品类**: B 端自用工具 / 经济决策引擎
**起源**: 2026-05-23

**关键学习点**:
1. **重大定位修正**: 从"补货 workflow 工具"重定位为"经济优化决策引擎"——价值从"省时间"变成"准确性"
2. **战略评审的金句**: "你教别人'战略就是选择不做什么'——这个项目本身就是这句话最好的试金石"
3. **多 Agent 辩论收敛**: 3 轮博弈后留 5 个 [OPEN_QUESTION] 给企业家拍板（含 ROI 拷问）
4. **协同写作的价值**: v0.2 R2 互相 review 揭示 35+ 条 v1.5 单 PM 写法发现不了的冲突

**最终成果**:
- v1.5 PRD (542 行) · 可执行性 370/600 = 62%
- v2 PRD (2499 行) · 可执行性 583/600 = 97%

**适合学什么**:
- 看 `debate-log.md` 学多 Agent 充分博弈
- 看 `drafts/r2-*` 学多角色互相 review 怎么找冲突
- 看 `assumptions.md` 学怎么把"猜测"显性化为可证伪假设
- 看 v1.5 vs v2 PRD 的对比学协同写作的真实价值
- 看战略评审 R3 的金句学怎么质疑"该不该做"

## 怎么用这些案例

### 学方法论

从一个跟自己项目相似的案例开始，按以下顺序看：
1. conversation.md（看企业家原始描述 + 全程对话）
2. scene-anchor.md（看场景怎么从模糊变清晰）
3. debate-log.md（看 3 轮博弈怎么把方案打磨）
4. PRD-v2.md（看终稿质感）
5. assumptions.md（看哪些是"上线后才能验证"的）

### 当模板

复制案例做参考：
```bash
cp -r prd-master/examples/01-fba-smart-restock/* projects/{your-project}/
```

但**不要**直接套——每个项目场景不同。参考结构，但内容必须重新跑。

### 反例学习

每个案例都暴露了 PRD 大师在某个版本的能力边界：
- 案例 01：暴露 v0.1 单 PM 写法的不足 → 催生 v0.2
- 案例 01 同时暴露 v0.2 Controller 合并的盲点 → 催生 v0.3 的 R4 跨章节 reviewer

跑案例时**主动找当前版本的不足**——这是改进 PRD 大师的最好方式。

## 怎么贡献新案例

跑完一个 PRD 项目后，如果质量好且有教学价值：

```bash
cp -r projects/{your-project} prd-master/examples/{NN-name}/
```

然后在本 README 加一段介绍。
