# 行动卡 / 进度卡格式规范（v0.3 P0-1）

## 为什么需要

v0.2 之前，Controller 输出是"被动模式"——回答完用户问题就停。企业家不知道：
- 现在到哪一步了？
- 还有多远？
- 接下来该做什么？

这是"被动咨询顾问"和"主动 PRD 大师"的根本差异。

## 强制格式

**Controller 每次给企业家的输出末尾，必须带一个"📍 行动卡"**。

格式：

```
---
📍 **当前进度**：阶段 X.Y · {阶段名}（已完成 N/6 阶段）

⏱ **本阶段还需**：约 {N} 分钟 / {M} 轮互动

👉 **你现在需要做的**：
{一句话告诉企业家此刻该做什么——回答一个问题 / 看一个文件 / 拍板一个选择}

🔮 **接下来会发生什么**（让企业家有预期）：
1. {下一步}
2. {再下一步}
```

## 6 阶段的预估时间和轮数（基准）

| 阶段 | 预估时间 | 轮数 |
|------|---------|------|
| 0 自由倾诉 | 1-3 分钟 | 1 轮 |
| 1 苏格拉底深挖 | 10-20 分钟 | 5-8 轮 |
| 2 自动调研 | 5-10 分钟 | 1 轮（系统自动跑） |
| 3 V0+辩论 | 15-30 分钟 | 3-5 轮 |
| 4 Demo（可选） | 10-20 分钟 | 1-2 轮 |
| 5 PRD 协同写作 | 20-40 分钟 | 系统自动跑 |
| **总计** | **60-120 分钟** | **10-20 轮** |

## 示例

### 阶段 0 结束

```

📍 **当前进度**：阶段 0.1 · 自由倾诉回放（已完成 1/6 阶段）

⏱ **本阶段还需**：< 1 分钟（你确认理解对不对）

👉 **你现在需要做的**：
看上面"我听到的是这样"，告诉我对不对、要补充啥

🔮 **接下来会发生什么**：
1. 进入阶段 1 苏格拉底深挖，我会问你 5-8 个关键问题（每问 1-2 个）
2. 把场景骨架挖清楚，约 10-20 分钟
3. 然后进调研、出方案、吵架、写 PRD
```

### 阶段 3.5 V1 方案出来后

```
（这里是 V1 方案 + debate 摘要）

---
📍 **当前进度**：阶段 3.5 · 方案 V1 出炉（已完成 3/6 阶段）

⏱ **本阶段还需**：你拍板 1 次（< 5 分钟）

👉 **你现在需要做的**：
看 V1 方案对不对——是直接走 PRD 终稿、再吵一轮、还是大方向调整

🔮 **接下来会发生什么**（取决于你的选择）：
- 直接走 → 进阶段 5 PRD 协同写作（20-40 分钟系统跑）
- 再吵一轮 → 回到 3.1 出 V0'，再走一遍博弈
- 大方向调 → 回到阶段 1 重新对齐
```

### 阶段 5 完成后

```
（这里是 PRD 交付清单）

📍 **当前进度**：阶段 5.6 · PRD 终稿交付（已完成 6/6 阶段 ✅）

🎉 **整个 PRD 流程完成**！总用时 X 分钟，N 轮互动。

👉 **你现在可以做的**：

4. 看 `debate-log.md`——这次跑通的所有博弈过程（教学素材）

🔮 **你接下来可能想做的**：
1. 直接把 `PRD.md` 喂给 Claude Code 开始写代码
2. 跟我说"看 OPEN_QUESTION"——我帮你过一遍待拍板事项
3. 跟我说"再做一个"——继续做下一个 PRD
4. 跟我说"我先想想，下次继续"——状态会保存到 state.json
```

## 行动卡 ≠ 详细日志

行动卡是给企业家看的**"此刻决策点"**，不是日志。

❌ 不要写"系统已经处理了 5 个 SKU 的销量数据..."
✅ 写"我已经把 200 SKU 的销量趋势画好，你拍板要不要继续调研物流"

## 行动卡的"主动感"原则

每张行动卡都要回答企业家潜意识的 3 个问题：

1. **"我现在在哪？"** → 当前进度
2. **"还要多久？"** → 本阶段还需
3. **"我该干啥？"** → 你现在需要做的

光有 1+3 是"咨询顾问汇报"，加上 2 才是"主动 lead 你"。
