## 2. 页面概述

### 2.1 一句话定位

> 用 1-3 句话说明这个页面**存在的唯一理由**。

**示例：** 让用户通过手机号 / 邮箱 / 验证码 / 第三方账号，完成安全、快速的登录。

### 2.2 存在理由

> 回答"没有这个页面会怎样"。列 1-3 条具体的风险或痛点。

**示例：**
1. 无门禁则任意人可进入系统，存在数据泄露风险
2. 无法识别用户身份，无法提供个性化内容
3. 无法记录用户行为，丢失学习数据

### 2.3 页面目标（可衡量）

> 用表格列出可衡量的目标指标。

**示例：**

| 目标维度 | 具体指标 |
| :--- | :--- |
| 登录成功率 | 正常路径登录成功率 ≥ 99% |
| 登录效率 | 用户从打开页面到登录完成 ≤ 15 秒 |
| 安全性 | 无有效登录态者 100% 被拦截 |

### 2.4 不做什么（范围边界）

> 列 3-5 项**看似相关但不属于本页面**的功能。每项注明"为什么不做"。这一节用于防止功能蔓延，是产品文档的重要边界。

**示例：**
- **不做**注册——注册是独立页面，本页仅负责登录
- **不做**密码找回/重置——跳转独立页面处理
- **不做**多角色选择——登录后由系统根据账号类型自动路由

| 使用设备 | 手机 / PC |

### 3.2 核心使用场景

> 至少写 2 个场景：1 个主场景 + 1 个兜底场景。每个场景包含"时机 / 前置条件 / 用户动作 / 用户情绪"。

**示例：**

**场景 A：首次登录（主场景）**
- 时机：用户刚注册成功 / 首次打开 App
- 前置：已有账号
- 动作：输入账号密码 → 点击登录 → 进入主页
- 情绪：期待、顺畅

**场景 B：密码错误后重试（兜底场景）**
- 时机：登录过程中密码记错 / 手误

- 动作：看到错误提示 → 修改密码 → 重新登录
- 情绪：不焦虑、可快速重试

### 3.3 用户核心诉求（优先级排序）

> 按优先级从高到低列出用户最在意的事。

**示例：**
1. **快**：一键到位，不要多余步骤
2. **准**：错了能立刻知道原因
3. **稳**：登录后不频繁掉线
4. **可选**：记住我，下次免输

## 6. 交互流程

### 6.1 页面加载与初始化

> 用户进入页面后看到什么，系统做了什么初始化。

**示例：**
- 检查是否已有登录态 → 已登录则直接跳转主页，不展示登录页
- 未登录 → 展示登录页，账号输入框自动聚焦

### 6.2 核心操作流程

> 用"用户做 X → 系统做 Y → 跳到 Z"的产品流程描述。不写动画、不写技术实现。

**示例：**

```
用户输入账号密码 → 点击"登录"
    │
    ├─ 账号或密码为空 → 提示"请输入账号和密码"，不发起请求
    │
    └─ 非空 → 系统校验身份
         │
         ├─ 校验失败 → 提示"账号或密码错误"，可重试
         │
         └─ 校验成功 → 建立登录态 → 跳转主页
```

### 6.3 辅助操作

> 快捷键、次要入口等。

**示例：**
- 密码框内按 Enter = 点击登录
- 切换密码显隐：点击眼睛图标
- 登录方式切换：点击 Tab

## 7. 状态与边界

### 7.1 页面状态全景

> 用表格列出页面所有状态。至少覆盖 6 态：初始 / 输入 / 提交 / 成功 / 失败 / 异常。

**示例：**

| 状态 | 触发条件 | 页面表现 |
| :--- | :--- | :--- |
| 初始态 | 页面首次加载 | 账号框聚焦，登录按钮可点击 |
| 输入态 | 用户正在输入 | 实时输入，无实时校验 |
| 提交态 | 点击登录后 | 按钮变为"登录中"，不可重复点击 |
| 成功态 | 校验通过 | 跳转主页 |
| 失败态 | 密码错误 | 提示"账号或密码错误"，可重试 |
| 异常态 | 网络超时 / 断网 | 提示"网络异常，请重试"，可重试 |

### 7.2 边界与容错

> 至少覆盖 5 种边界场景。这是产品文档的核心价值——防漏 Case。

**示例：**

| 边界场景 | 处理策略 |
| :--- | :--- |
| 连续密码错误 | 不锁死。连续错误 ≥ 5 次，温和提示"密码多次错误，请确认" |
| 网络断开 | 提示"网络异常"，用户可直接重试，无需刷新 |
| 会话过期 | 跳回登录页，副标题可提示"登录已过期，请重新登录" |
| 超长输入 | 前端不限制，后端截断保护 |
| 粘贴密码 | 允许粘贴（用户可能从备忘录复制） |
| 浏览器自动填充 | 允许，不主动拦截 |

## 8. 业务规则与校验

### 8.1 字段定义

> 定义页面涉及的数据字段。

**示例：**

| 字段 | 说明 |
| :--- | :--- |
| 账号（account） | 手机号或邮箱，用户唯一标识 |
| 密码（password） | 至少 8 位，含数字和字母 |
| 验证码（code） | 6 位数字短信验证码 |

### 8.2 校验规则

> 用表格列出每个校验项：时机 / 规则 / 失败反馈文案。

**示例：**

| 校验项 | 时机 | 规则 | 失败反馈 |
| :--- | :--- | :--- | :--- |
| 账号非空 | 提交时 | trim 后不为空 | "请输入账号" |
| 账号格式 | 失焦时 | 手机号或邮箱格式合法 | "账号格式不正确" |
| 密码非空 | 提交时 | trim 后不为空 | "请输入密码" |
| 密码格式 | 提交时 | ≥ 8 位，含数字和字母 | "密码至少 8 位，含数字和字母" |
| 身份校验 | 后端 | 账号密码匹配 | "账号或密码错误" |
| 验证码非空 | 提交时 | 6 位数字 | "请输入验证码" |

### 8.3 特殊业务规则

> 防滥用、登录态、权限等特殊规则。

**示例：**
- **防重复提交**：提交期间按钮禁用
- **验证码倒计时**：点击获取后 60 秒内不可重复获取
- **登录态时效**：建议 ≥ 12 小时，避免频繁重登
- **记住我**：勾选后本地保存账号（不存密码），下次自动填充
