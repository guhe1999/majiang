# 提示词 · 07 页面 HTML 开发师

> ## ⚠️ 执行方式（最高优先级 · 必须遵守）
>
> **本提示词为「单页」生成逻辑。必须并发调用子代理执行，每个子代理负责一个页面 HTML：**
>
> 1. 主代理先读取 `output/页面清单.md`（或扫描 `output/pages/*.md`）获取全量页面列表
> 2. **按系统支持的最大并发数并发派发**（不要一次性派发全部 M 个——并发过多会超出系统上限失败）：维持满并发，每完成一个立即补派下一个，直到全部页面 HTML 完成；每个子代理负责一个页面 HTML，收到「阶段 + 功能模块 + 页面名称」，各自从对应 `.md` + `DESIGN.md` + `tokens.css` 生成

> 4. 全部完成后，主代理再做一次全局自校验（`:root` 外 hex / tokens.css 一致 / 文件齐全）
>
> **串行逐页生成、用一个子代理生成多个页面 HTML、或一次性派发全部超出系统并发上限，均视为违规。**

> **使用方法**：把以下全部内容复制，粘贴到 Claude Code 对话框发送即可。
>

> - `output/DESIGN.md` 已存在（由 02-设计系统设计师 产出）

# Implementation Requirements / 实现要求

## 1. Tech Stack & Resources（技术栈与资源）

- **核心技术**
  - 使用原生 **HTML + Tailwind CSS** 实现页面。
  - 禁止引入任何第三方 UI 框架（如 Bootstrap、Ant Design、Material UI 等）。

- **Head 区资源引入（必须）**
  在 `<head>` 中至少包含以下内容：
  - Tailwind CSS CDN（仅用于布局工具类：flex、grid、spacing、responsive）：
    ```html
    <script src="https://cdn.tailwindcss.com"></script>
    ```
  - 字体（从 tokens.css 的 `--font-display` / `--font-body` 中提取）：
    - 例如通过 Google Fonts 引入：
      ```html
      <link rel="preconnect" href="https://fonts.googleapis.com">
      <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
      <link href="https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600;700&display=swap" rel="stylesheet">
      ```
  - 图标库（必须使用真正的矢量图标，而非文字代替图标）：
    - 建议使用 **Phosphor Icons**：
      ```html
      <script src="https://unpkg.com/@phosphor-icons/web"></script>
      ```

- **设计 Token 注入（关键要求）**
  读取 `tokens.css`，将其 `:root {}` 块（及 `[data-theme="dark"]` 块，如有）**原封不动粘贴** 到页面的 `<style>` 中：
  ```html
  <style>
    /* ===== 从 tokens.css 粘贴 ===== */
    :root {
      --bg: #fafafa;
      --surface: #ffffff;
      --accent: #2f6feb;
      /* ... tokens.css 中的全部变量 ... */
    }
    /* ===== end tokens ===== */
  </style>
  ```
  此后全文使用 `var(--xxx)` 引用，**禁止硬编码 Hex 颜色值**：
  - ✅ `color: var(--fg);`
  - ✅ `background: var(--accent);`
  - ✅ `border-radius: var(--radius-md);`
  - ❌ `color: #111111;`
  - ❌ `background: #2f6feb;`

  组件 CSS 优先从 `DESIGN.md` §6 组件设计中**直接复制**，动效 CSS 从 §7 动效与交互物理中**直接复制**，保持与设计系统一致。

* **自定义 CSS**

  * 如需实现少量特殊效果（如玻璃拟态、柔和渐变边框、滚动条优化等），可以在 `<style>` 中补充 **极少量** 自定义 CSS。
  * 禁止大规模重写 Tailwind 默认样式，避免破坏整体风格的一致性和可维护性。

## 2. Page Scope（页面范围）

* 本次只实现 `页面内容.md` 中指定的 **单个页面**（会给出具体页面编号与名称）。
* 页面必须完整覆盖该页面描述中的 **全部核心信息与模块**，包括但不限于：
  * 顶部导航 / 面包屑；
  * 主功能区域（核心任务、关键操作、主要内容展示）；
  * 辅助说明、引导文案、提示信息；
  * Tab / 过滤器 / 视图切换；
  * 列表或卡片内容区域；
  * 概览统计 / 数据卡片；
  * 表单字段（输入框、选择器、开关、搜索框等）；
  * 关键操作的即时反馈（如提交成功后的提示）；
  * 标签 / 徽标 / 状态 Badge 等。
* 若 `页面内容.md` 中提供了示例文案或字段名，你需要在页面中 **完整体现**。
* 如有信息空缺，你需要根据该页面的业务语境，**主动补充合理的 mock 文案和数据**，但不得改变原有的信息结构和层级。

## 3. Layout & Responsiveness（布局与响应式）

* **桌面端优先（Desktop-first）**
  * 以宽度约 **1440px** 的桌面视口作为主设计参考。
  * 页面主体内容宽度建议在 **1080–1200px** 之间，并在视口中 **水平居中**。
  * 顶部、内容区、底部的垂直空间要均衡，**避免内容挤在页面上方导致底部大片空白**。
  * 若为仪表盘或概览类页面，建议使用 **Bento Grid（便当盒布局）**。

* **多栏布局**
  * 根据 `页面内容.md` 中的信息架构合理选择：单列 / 主内容+侧边栏 / 2-3列卡片栅格。
  * 不允许简单堆砌模块，必须体现出清晰的信息优先级与视觉层级。

* **移动端适配（约 375px 宽度）**
  * 移动端采用单列纵向滚动布局。
  * 核心任务区域应放在页面前部。
  * 确保：字号可读、按钮高度与点击区域充足、组件间间距合理。

* **共同要求**
  * 使用 Tailwind 的响应式前缀（sm/md/lg/xl）精细控制各断点布局。

* **基础交互结构**：使用少量内联 `<script>` 实现 class 切换，无需真实数据提交，但需要完整交互结构与状态反馈。

* **可访问性与语义化**：合理使用语义化标签和 aria 属性。

## 6. Content & Mock Data（内容、文案与 Mock 数据）

* **只展示正常态（典型使用中的完整数据）**。原型是让业务人员看懂"这个页面干嘛的"，不是状态展示页。
  - **不要**堆砌空状态、错误态、加载态——这些是开发实现时的细节，不该出现在高保真原型里干扰页面核心信息。
  - 唯一例外：如果"空状态"本身就是页面的核心设计（如"待办列表为空"的引导界面），才展示。
* **Mock 数据要求**：
  - 列表型模块：至少 **10 条** 示例数据。
  - 统计/概览型模块：具体、合理的数值。
  - 详情型内容：正文不少于几个段落。
* **禁止占位符内容**：严禁 Lorem Ipsum、"Item 1"、"TODO"等。
* **禁止在页面上写设计理念/思考过程**：页面是给业务人员看的高保真原型，不是设计说明文档。不要在页面上出现"本页面采用XX设计理念""此处体现XX原则"等说明性文字。

## 输出后自校验（每个页面生成后必须执行）

生成 HTML 并保存后，**立即**对自己刚写的文件执行以下检查：

1. **Grep 搜索 `:root {`**：确认 `<style>` 中包含从 tokens.css 粘贴的 `:root {}` 块
2. **Grep 搜索 `#[0-9a-fA-F]{3,6}`**：检查硬编码 hex 颜色。hex 值只允许出现在 `:root {}` 块内部，组件 CSS 中出现任何 hex 值即为违规
3. **Grep 搜索 `#6366f1|#4f46e5|#8b5cf6|#7c3aed|#a855f7`**：检测 Tailwind 默认靛蓝/紫色——这是 AI 生成设计的第一特征。如果出现，说明没有走设计系统的 `--accent`，必须替换为 `var(--accent)`
4. **Grep 搜索 emoji**（`✨|🚀|🎯|⚡|🔥|💡`）出现在 `<h`、`<button`、`<li` 标签中：emoji 当功能图标是 AI 味的典型信号，必须替换为 SVG 图标
5. **如果检查不通过**：自行修复，修复后重新保存，禁止交付不合规的 HTML

> 这不是可选步骤。每个 HTML 文件保存后都必须执行，通过后才能汇报"已完成"。
