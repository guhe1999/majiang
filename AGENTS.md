# Codex 协作约定

## 项目基线

- 本项目是地方性棋牌游戏平台，当前优先开发青岛麻将“推倒胡”。
- 产品范围和模块边界以 `outline.md` 为准。
- 开始任务前先阅读 `CONTEXT.md`、`HANDOFF.md` 和本任务涉及的源码或文档。
- 不凭空补充玩法规则；不确定的规则记录到 `HANDOFF.md` 的“待确认事项”。

## 分支与同步

- 家里和公司的 Mac 均使用 `jiahe` 作为持续开发分支。
- 开始工作前执行 `git switch jiahe` 和 `git pull --rebase origin jiahe`。
- 提交前检查 `git status`，不得提交密钥、账号、`.env` 或操作系统临时文件。
- 一次提交只包含一个清晰目标，提交信息使用简明中文。
- 未经用户明确要求，不向 `main` 合并或推送。

## Codex 任务交接

- `CONTEXT.md` 保存稳定背景、架构边界和长期决策，不记录流水账。
- `HANDOFF.md` 保存当前状态、最近完成、下一步和阻塞项。
- 每次开发结束前，Codex 必须更新 `HANDOFF.md`，使另一台 Mac 的新任务能够继续工作。
- 新设备或新任务中，先根据上述文档恢复上下文，再修改代码。
- 聊天中的关键决策必须落入 `CONTEXT.md` 或相关项目文档，不能只存在于聊天记录中。

## 验证

- 修改解析工作台相关内容后，运行 `node script/validate-goal.mjs`。
- 若无法执行验证，在 `HANDOFF.md` 中写明原因和未验证范围。
- 不删除或覆盖用户未提交的改动；发现无关改动时保留并说明。
